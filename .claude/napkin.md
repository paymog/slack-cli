## Patterns That Work
- `provider.New(transport, logger)` reads tokens from `SLACK_MCP_*` env vars. CLI resolves a profile, sets those env vars in-process (`config.ApplyToEnv` via os.Setenv), then calls `provider.New`. Zero provider changes.
- Cache: `RefreshUsers/RefreshChannels(ctx)` load disk cache synchronously + set ready (network-fetch synchronously only on first run/missing cache). `ForceRefreshUsers/Channels(ctx)` = synchronous network fetch + disk write â†’ use for `cache refresh`. `SkipCache()` marks ready w/o loading â†’ use for `--no-cache`.
- Conventions copied from `~/code/groundcover-cli` (Go, cobra, zalando/go-keyring, profiles.yaml via os.UserConfigDir, `auth login/list/default/status/token/logout`, `internal/{cli,config,credstore,output}`, ldflags version in `internal/cli.version`, goreleaser + homebrew tap `paymog/tap`, ships a `skills/<name>-cli/SKILL.md`).
- **CLI output is JSON, not CSV** (PR #2, `feat/json-output`). `internal/output.Print(w, text, raw, tabular)`: valid JSON â†’ re-indented (order preserved); when `tabular` â†’ CSV parsed to a JSON array of string-keyed objects (column order kept, HTML escaping OFF so `<@U>`/`&` stay legible); else verbatim. Tabular commands call `emitTable` (channels list/me, users search, conversations history/replies/search/unreads, saved list, usergroups list/me); the rest use `emit`. `--raw` = original CSV/text. **Conversion is CLI-layer ONLY** â€” `pkg/handler` still marshals CSV byte-for-byte â†’ clean upstream merges + MCP server output unchanged. Guard: a single-column "table" (gocsv always emits â‰Ą2 cols) is a status line ("No users found.") â†’ passed through, never converted. gocsv headers for `Message`/`Channel` = Go field NAMES (PascalCase `Text`,`MemberCount`,`Cursor`), because those structs carry json tags but no csv tags â†’ JSON keys are PascalCase (jq: `.[].Text`, not `.text`).

PR #2 squash-merged to `master` (`ed6ea66`); `make release TAG=v0.2.0` â†’ release.yaml succeeded (4 tarballs + checksums on GH release, `paymog/homebrew-tap` formula updated). Merge needs squash; PR auto-targets `upstream` (korotovsky) â€” always `gh pr create --repo paymog/slack-cli --base master`.
- **Image attachments return bytes out-of-band** â€” upstream (`eb4d63b9`, PR #237) makes `FilesGetHandler` emit image files via `mcp.NewToolResultImage` = a `TextContent{metadata-only JSON}` + a SEPARATE `ImageContent{base64 Data}`. The original `toolcall.Invoke` extracted only `TextContent`, so `attachments get <image>` silently dropped the bytes (non-image binaries were fine: single `NewToolResultText` with `{...,"encoding":"base64","content":"<b64>"}`). Fix is **CLI-layer ONLY** (handler stays byte-for-byte upstream): `toolcall.InvokeResult` returns `Result{Text string; Images []Blob{Data,MIMEType}}` capturing `ImageContent`; `Invoke` is now a thin wrapper returning `.Text` (still drops images, existing callers unchanged). `attachments get` gained `-o/--output <path>` (decode bytes â†’ file, stdout = small metadata confirmation JSON, no blob â€” good for agents) and without `-o` folds the image bytes back into the same base64 envelope as non-image binaries (parity â†’ `jq -r .content | base64 --decode > f`). Helpers live in `internal/cmds/attachments.go` (`writeAttachment`/`attachmentBytes`/`injectContent`); `injectContent` string-appends `encoding`+`content` to preserve field order (a struct round-trip would sort keys + HTML-escape Slack markup).

## Gotchas (self)
- **Write/sensitive handlers self-gate INSIDE their parse funcs** (`parseParamsToolAddMessage`/`Reaction`/`Mark`/`FilesGet` each check `SLACK_MCP_ADD_MESSAGE_TOOL`/`_REACTION_TOOL`/`_MARK_TOOL`/`_ATTACHMENT_TOOL`). The CLI inherits the guards for free â€” no CLI-side gating needed; user must set the env var to use `conversations add`/`reactions`/`conversations mark`/`attachments get`. (Server registration in server.go ALSO gates, but the self-gate is what protects the CLI.)
- `provider.New` calls `logger.Fatal` (os.Exit) on missing/invalid auth. Pre-check `config.RequireAuth()` for a clean error; build a quiet zap logger (ErrorLevel) so Info logs ("Authenticated to Slack", "Loaded users from cache") don't spew. `--verbose` raises level.
- Handler param TYPES differ: `conversations` reads `limit` as **string** (`GetString`, "1d"); `channels` reads `limit` as **int** (`GetInt`). Build the args map with the type each handler expects.
- mcp-go is `v0.44.0`. `CallToolRequest{Params: CallToolParams{Name, Arguments: map[string]any}}`; `GetArguments()` requires `Arguments` to be `map[string]any`. Result text = concat `res.Content[].(mcp.TextContent).Text`; honor `res.IsError`. `toolcall.InvokeResult` additionally captures `mcp.ImageContent` (`.Data` base64 + `.MIMEType`) into `Result.Images`; `Invoke` still drops non-text content.
